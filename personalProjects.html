This was a project to help keep skills that I recently learned sharp and to apply them to something I had genuine curiosity about.

To those who do not know, Team Fortress 2 (TF2), is a first person shooter created by Valve in 2007 and is host to a number of maps. A lot of these maps are actually made by members of the community, and is still updated (somewhat) to this day. I had a few questions I wanted to ask that seemed best to answer through data analysis and as such, begun working.

First step of the project was data collection, I got majority of data from official TF2 wiki page, which has every map listed there. I also followed a process labeled by Youtuber Uncle Dane on how to get the rough playable space of all of these maps and created my own data set to work with.

After gathering all of the data, I put it all into Python, and begun to create various charts and visualizations to show off my findings. At first I was creating the charts using matplotlib, but decided to move to Dash and Plotly as those would allow me to put my findings on a website and still retain interactability.

You can access the completed project with fully interactable charts by using the following link:

https://tf2mapanalysis2023.onrender.com

*Please note that due to the hosting system used, the page can take up to 2 minutes to initially load, please give it some time if the site gives you a blank page at first

This was an idea I had gotten while learning how to visualize data in Python. Using the Wordcloud library, you can use a black and white image (Or really just any image that is just a single color with a empty background) as a mask for the word cloud to fit inside of.

The core idea of the project was to take every frame from the music video Bad Apple, which is already in black and white, and apply a word cloud to every frame. From here, I can stitch the frames back together and render it back into a proper video.

The end result was 6,953 word cloud images being created, all of which were made by the program I created running a single time.

The website you are currently seeing has actually gone through a few iterations. I actually originally used the same website layout I had used for my final project in the Web Design class in college, but decided to create a new design now that I have more time on my hands.

A goal I had when creating the site was to have everything appear under the main outline. This means that I could not allow the entire page to scroll, leaving me with the option to have just a single section of a 3 column layout available for scrolling.

One thing that actually took a while was trying to get the general layout of the site down. I ended up having to make a test version with more distinct colors in each section just so I can better troubleshoot the thousands of issues I ended up creating along the way.

One thing I had got working on an older version of the site was a short animation that would play when your first enter the site. It was to resemble an old TV turning on. While it was mostly functional, there were some minor issues with alignment that needed consideration. I would also have needed to put in a way to prevent the animation from playing when selecting the links on the side (Otherwise it would appear every time you changed page, which would be highly annoying).

The website itself is still a in-progress thing, as I will slowly over time add more to it, fix the many issues with its layout and update it will the various projects I have worked on. I already know that I plan on working on a more mobile friendly version of the site, and creating a system that should allow it to switch between mobile and desktop version automatically.

As with any website, this one was created using a mixture of HTML, CSS, and JavaScript.

As you likely already know, you can press F12 to open up the website source on your browser, which can allow you to view the code. Feel free to look around and witness the very strange way I went about making this site.

A project I decided to work on in high-school purely out of boredom one day. The idea was to replicate the style of game found in the zombies mode of Call Of Duty, but put it in a top down perspective (ignoring the fact that Call Of Duty has actually done this...sorta).

A main goal was to create a system that would both slowly increase the amount of zombies that spawn, while also making them more durable as the waves increases.

A major challenge I had to overcome was figuring out how to allow zombies to spawn in areas that the player just unlocked, and while the solution was not elegant, it works.

The game was made in GameMaker Studio, and while I never did get around to properly finishing it, I am still proud of the progress made on it.

When it comes to certain video games, they have the ability to be edited by the user. This is usually referred to as modding.

Examples of games that I had created mods for are: Don't Stare (Both the singleplayer version of the multiplayer version), Spelunky, Noita, and VRChat (Sorta)

In two of the four examples, the mods themselves are rather simple, they are just replacing one image with another, but the process of doing so can take a surprisingly large amount of time (and in the case of the Spelunky mod, I cheated by just recoloring an image)

In the case of Don't starve, the mod itself allows for each character to have different abilities. Most characters would usually have a mixture of positive and negative quirks to keep things balanced. In my mod, I made it so the player would passively regen their sanity stat, but also loose it at a faster rate

VrChat is an odd case of modding. As it currently stands, modding the actual game goes against their terms of service. However, they have a system in place that allows users to create and upload their creations directly onto the platform. This will either be in the form of avatars or worlds.
